E-commerce Redesign

person Lead Developer

문제 정의 (Problem)

기존 플랫폼은 복잡한 체크아웃 과정으로 인해 모바일 전환율이 경쟁사 대비 15% 낮았으며, 레거시 코드베이스로 인해 새로운 결제 수단 추가 시 시스템 안정성이 저하되는 고질적인 문제를 안고 있었습니다.

해결 과정 (Solution)

bolt

Architecture Shift

모놀리식 구조에서 Next.js 기반의 서버 사이드 렌더링(SSR) 아키텍처로 전환하여 초기 로딩 속도를 40% 개선했습니다.

payments

One-Tap Checkout

복잡한 입력 폼을 간소화하고 생체 인증 결제를 도입하여 사용자 여정을 5단계에서 2단계로 단축했습니다.

성과 및 지표 (Results)

+24% 전환율 상승
-1.2s 로딩 시간 단축
99.9% 결제 성공률
3x 모바일 매출
